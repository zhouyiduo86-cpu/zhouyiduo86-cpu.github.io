Case Study: Lifecycle Wage Dynamics & Inequality

Executive Summary & Deliverables

This labor economics study examines how the gender wage gap and college premium evolve across the worker lifecycle using IPUMS CPS microdata. An R-based pipeline cleans and trims 200,000+ observations, then estimates baseline and interaction-rich Mincer regressions to map non-linear earnings trajectories and decompose wage inequality into hours-worked and hourly-rate components.

View Full Analysis Report (PDF) View R Script

Microdata Structure (IPUMS CPS)

The analysis ingests compressed CPS extract data, applies sample restrictions for prime-age full-time workers, and constructs derived wage measures for regression and visualization.

Variable Data Type Description
INCWAGE Numeric Total annual wage and salary income (dependent variable source)
AGE Integer Worker age (restricted to 25–60 for lifecycle analysis)
EDUC / educ5 Categorical Education level recoded into five ordered categories
SEX Binary Gender indicator for wage-gap decomposition
hourly_wage Derived Float INCWAGE divided by annual hours (robustness measure)

Econometric Modeling (R)

Core data pipeline and Mincer regression specification from wage_dynamics_model.R — including interaction terms for age, education, and gender. 

library(dplyr)
library(ggplot2)
library(tidyr)
library(broom)

data <- read.csv("cps_00002_csv.gz")

df_clean <- data %>%
  filter(AGE >= 25, AGE <= 60) %>%
  filter(!is.na(INCWAGE), !is.na(UHRSWORKLY), !is.na(WKSWORK1),
         !is.na(AGE),     !is.na(EDUC),        !is.na(SEX)) %>%
  filter(INCWAGE >= 2200) %>%
  mutate(
    annual_hours = UHRSWORKLY * WKSWORK1,
    hourly_wage  = INCWAGE / annual_hours
  ) %>%
  filter(!(hourly_wage > 450 & annual_hours < 100))

df_trim <- df_clean %>%
  filter(INCWAGE >= quantile(INCWAGE, 0.01),
         INCWAGE <= quantile(INCWAGE, 0.99)) %>%
  mutate(
    logwage  = log(INCWAGE),
    loghwage = log(hourly_wage),
    educ5 = case_when(
      EDUC == 73                ~ "HS",
      EDUC %in% c(81,91,92,111) ~ "SomeCollege",
      EDUC == 123               ~ "College",
      EDUC %in% c(124,125)     ~ "Grad",
      TRUE                      ~ "ltHS"
    ),
    educ5 = relevel(factor(educ5), ref = "ltHS")
  )
